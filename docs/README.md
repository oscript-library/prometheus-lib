# Спецификация: промежуточный формат и сериализация Prometheus Text Format

Спецификация реализации на OneScript: библиотека формирует данные в **промежуточном формате** и передаёт их в
функцию сериализации; результат должен совпадать для одних и тех же входных данных.

## 1. Промежуточный формат (результат Gather / Собрать)

Данные — **массив семейств метрик**. Каждое семейство — структура с полями:

| Поле        | Тип           | Описание                                  |
|-------------|---------------|-------------------------------------------|
| **Имя**     | Строка        | Имя метрики (напр. `http_requests_total`) |
| **Тип**     | Строка        | counter, gauge, histogram, summary        |
| **Справка** | Строка (опц.) | Текст для `# HELP`                        |
| **Сэмплы**  | Массив        | Список сэмплов (см. ниже)                 |

### Сэмпл (элемент массива Сэмплы)

Структура с полями:

| Поле         | Тип                         | Описание                 |
|--------------|-----------------------------|--------------------------|
| **Лейблы**   | Соответствие (имя - знач.)  | Лейблы для `name{...}`   |
| **Значение** | Число                       | Числовое значение сэмпла |

Для **histogram** у сэмплов с суффиксом `_bucket` дополнительно:

| Поле               | Тип              | Описание                            |
|--------------------|------------------|-------------------------------------|
| **ВерхняяГраница** | Строка или Число | Лейбл `le`; последний бакет: "+Inf" |

Для **summary** у сэмплов с квантилями дополнительно:

| Поле         | Тип   | Описание                                       |
|--------------|-------|------------------------------------------------|
| **Квантиль** | Число | Значение лейбла `quantile` (например 0.5, 0.9) |

### Имена сэмплов для histogram и summary

- **Histogram:** сэмплы с именем `ИмяМетрики_bucket` (лейбл `le`), один сэмпл `ИмяМетрики_sum`, один
  `ИмяМетрики_count`. В промежуточном формате имя семейства — базовое (без _bucket/_sum/_count); суффикс —
  тип сэмпла или поле **СуффиксИмени** (`""`, `"_bucket"`, `"_sum"`, `"_count"`).
- **Summary:** сэмплы с полем Квантиль — имя как у семейства; сэмплы `_sum`/`_count` — как у histogram.

Для упрощения реализации допускается передавать сэмплы с **полным именем** (например
`http_request_duration_seconds_bucket`), тогда Лейблы содержат `le`, а в семействе Имя может быть базовым.
Сериализатор группирует по базовому имени (до _bucket/_sum/_count) и выводит один блок HELP/TYPE на семейство.

**Рекомендация:** в семействе хранить **Имя** базовое (без суффикса); каждый сэмпл содержит полное имя в
**ИмяСэмпла** (опц.) или по правилам: counter/gauge — Имя семейства; histogram — Имя + _bucket/_sum/_count;
summary — Имя или Имя + _sum/_count. Либо все сэмплы имеют **ИмяСэмпла**, сериализатор использует его.

**Минимальный вариант для сериализатора:** на вход массив семейств; семейство: Имя, Тип, Справка, Сэмплы;
сэмпл: Лейблы, Значение, опц. ВерхняяГраница (le), Квантиль (quantile), ИмяСэмпла (если ≠ Имя семейства,
иначе Имя + суффикс по типу).

## 2. Правила сериализации (Prometheus text format)

- Кодировка: **UTF-8**. Перевод строки: **\n**. Последняя строка заканчивается **\n**.
- Для каждого семейства метрик (в лексикографическом порядке по имени):
  1. Если есть Справка: строка `# HELP имя_метрики справка`
     (экранирование в справке: `\` → `\\`, `\n` → `\n`).
  2. Строка `# TYPE имя_метрики тип` (тип: counter, gauge, histogram, summary).
  3. Для каждого сэмпла (порядок: см. ниже): строка `имя_сэмпла{лейблы} значение`
     или `имя_сэмпла{лейблы} значение timestamp` (timestamp опц., миллисекунды).
- **Экранирование в значениях лейблов:** `\` → `\\`, `"` → `\"`,
  перевод строки → `\n`.
- Имена метрик и лейблов: только допустимые символы ([a-zA-Z0-9_:]);
  иначе кавычки по спецификации Prometheus.
- **Histogram:** сэмплы _bucket — по возрастанию le (числа, затем "+Inf");
  затем _sum; затем _count.
- **Summary:** сэмплы с quantile — по возрастанию квантиля; затем _sum; затем _count.

### Content-Type для HTTP

```txt
text/plain; version=0.0.4; charset=utf-8
```

## 3. Пример выходной строки

```txt
# HELP http_requests_total Всего HTTP-запросов.
# TYPE http_requests_total counter
http_requests_total{method="get",code="200"} 1027
http_requests_total{method="post",code="400"} 3
```

## 4. Реализация в OneScript

Функция сериализации принимает массив семейств (каждый — структура с полями Имя, Тип, Справка, Сэмплы) и
возвращает строку в формате Prometheus. Тесты проверяют, что один и тот же набор входных данных даёт
идентичную выходную строку.
